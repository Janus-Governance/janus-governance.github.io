What is Janus

Traceable governance for AI-assisted systems

Canonical Architecture

Overview

Janus is a governance core for systems where humans, software, and AI participate in decision processes.

Janus does not execute business logic. It governs how system evolution is evaluated, recorded, and reconstructed.

The problem

Most systems record what happened. Very few systems record what should have happened but did not.

This becomes critical in AI-assisted environments, where omissions and missing validations can create risk.

The Janus approach

Janus evaluates governance conditions rather than application logic. Key ideas include the evidence model (E+ / E−), governance events, human authority, the audit writer boundary, and deterministic rebuild. 

activity → evaluation → governance event → audit log → rebuildable state

System map

The Janus architecture is a strict, layered system. Each layer depends only on those above it. Demos and Adapters are shown in their canonical positions.

RFCs (Formalization Layer)
   ↓
Core (Normative Layer)
   ↓
Framework (Operational Layer)
   ↓
Runtime(s) (Execution Layer)
   ↓
SDK / Tools (Developer Layer)
   ↓
Implementations (Application Layer)

Adapters (Integration Boundary Layer) ←→ Runtime(s)
Demos (Validation Artifacts) [outside main execution path]
  • RFCs: Formalize, evolve, and constrain the Core.
  • Core: Defines governance semantics, contracts, and invariants.
  • Framework: Reusable governance mechanisms implementing Core semantics.
  • Runtime(s): Executes governed evaluation.
  • SDK / Tools: Integration-facing utilities and developer surfaces.
  • Implementations: Real-world systems using Janus.
  • Adapters: Connects external systems to Janus semantics.
  • Demos: Non-authoritative demonstration systems.
